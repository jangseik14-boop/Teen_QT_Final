
# 🎉 예본TeenQT 배포 긴급 해결 가이드

빌드 에러(`exit status 1`)가 발생했다면, 구글 서버의 메모리 설정이나 보안 정책 때문일 확률이 99%입니다. 제가 방금 이 문제를 해결하는 코드를 새로 올렸습니다.

### 🚀 해결 순서 (필독!)
1. **[출시 만들기] 누르기**: Firebase 콘솔 상단의 파란색 **[출시 만들기]** 버튼을 다시 한번 눌러주세요.
2. **브랜치 입력**: 팝업창이 뜨면 브랜치 이름에 **`main`**을 입력하세요.
3. **설정 완료**: '기본 브랜치로 설정'에 체크하고 **[만들기]**를 누르세요.
4. **5분만 기다리기**: 이번에는 메모리가 2048MiB로 넉넉해져서 에러 없이 끝까지 갈 거예요.

### 🛠️ "백엔드를 찾을 수 없습니다" 해결법
- 배포가 끝나고 처음 1~2분은 서버가 깨어나는 시간입니다.
- 이 화면이 보인다면 **새로고침(F5)**을 3~4번 반복해 보세요.

### 💡 관리자 정보
- **비밀번호**: `141414`
- **위치**: `MY` 탭 최하단 [관리자 모드]

이제 다시 **[출시 만들기]**를 누르고 **`main`**을 입력하시면 정상적으로 배포될 거예요! 🤘
