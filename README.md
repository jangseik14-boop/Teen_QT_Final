
# 예본TeenQT 배포 가이드

학생들이 사용할 수 있도록 앱을 배포하는 상세 방법입니다.

## 1. GitHub에 코드 올리기 (중요: 이 단계를 완료해야 배포 버튼이 활성화됩니다)
컴퓨터의 터미널에서 다음 명령어를 한 줄씩 입력하세요:
```bash
# 1. 변경된 모든 파일 선택
git add .

# 2. 메시지와 함께 저장
git commit -m "배포 준비 완료"

# 3. GitHub 서버로 전송 (이걸 해야 Firebase에서 인식을 합니다)
git push -u origin main
```

## 2. Firebase App Hosting 설정 (버튼이 안 눌릴 때)
- **증상**: "브랜치 이름은 선택한 저장소의 유효한 기존 브랜치를 참조해야 합니다"라는 빨간 메시지가 뜨는 경우.
- **원인**: 위 1번 단계(`git push`)를 하지 않았거나, GitHub 서버에 반영되는 데 시간이 조금 걸리는 경우입니다.
- **해결**:
    1. GitHub 사이트에서 파일이 잘 올라왔는지 확인합니다.
    2. 파일이 보인다면, Firebase 페이지를 **새로고침(F5)** 합니다.
    3. 다시 `main`을 입력(또는 선택)하면 **[다음]** 버튼이 활성화됩니다.

## 3. 환경 변수 등록 (AI 작동 필수!)
배포 완료 후, App Hosting 대시보드 -> Settings -> Environment variables에서 다음을 추가하세요.
- **Variable**: `GEMINI_API_KEY`
- **Value**: 발급받은 API 키 (Secret으로 등록 권장)

## 4. 라이브 브랜치 정보
- **브랜치 이름**: `main`
- **앱 루트 디렉터리**: `/`
