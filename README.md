
# 🚀 예본TeenQT 배포 완료 후 가이드

축하합니다! 앱 배포가 시작되었습니다. 이제 AI 기능을 활성화하기 위한 마지막 설정을 진행해 주세요.

## 1. AI API 키 등록 (필수!)
배포가 완료되어도 이 설정을 하지 않으면 말씀 해설 AI가 작동하지 않습니다.

1.  Firebase 콘솔의 **App Hosting** 메뉴로 이동합니다.
2.  생성된 백엔드(`teen-qt-backend`)를 클릭합니다.
3.  상단 탭에서 **설정 (Settings)**을 선택하고 왼쪽 메뉴에서 **환경 (Environment)**을 누릅니다.
4.  **환경 이름** 칸에 `prod` 라고 입력하고 반드시 우측의 **[저장]** 버튼을 먼저 누르세요.
5.  **[중요]** 저장을 누른 후 화면에 아무것도 안 나타난다면, **페이지를 새로고침(F5)** 하거나 **화면을 아래로 스크롤**해 보세요.
6.  `prod` 환경 아래에 나타나는 **환경 변수 (Environment variables)** 섹션에서 **[Add variable]**을 클릭합니다.
7.  아래 정보를 정확히 입력하고 저장하세요:
    *   **이름(Key)**: `GEMINI_API_KEY`
    *   **값(Value)**: Google AI Studio에서 발급받은 API 키
8.  저장 후, 설정이 적용되도록 잠시 기다리거나 대시보드에서 '다시 배포'를 누르시면 됩니다.

## 2. 학생들에게 주소 공유하기
- 배포가 성공하면 App Hosting 대시보드 상단에 `https://...hosted.app`으로 끝나는 주소가 생깁니다.
- 이 주소가 우리 청소년부 친구들이 접속할 **공식 주소**입니다! 이 링크를 복사해서 카톡방에 공유해 주세요.

## 3. 관리자 모드 접속
- 선생님이나 전도사님 계정으로 로그인한 뒤, **MY** 탭에서 **관리자 모드**를 누르세요.
- 비밀번호 `141414`를 입력하면 학생들의 묵상 현황과 상품 신청을 관리할 수 있습니다.

## 4. 문제 해결 (로그인이 안 될 때)
- Firebase 콘솔의 **Authentication** 메뉴에서 **Settings > Authorized domains**에 현재 배포된 도메인(`...hosted.app`)이 추가되어 있는지 확인해 주세요. (보통 자동으로 추가됩니다.)
