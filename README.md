---

## 🗑️ [완전 초기화] 백엔드 삭제하고 다시 하기 (최후의 수단)
설정이 꼬여서 도저히 안 될 때는 삭제하고 다시 만드는 게 제일 빠릅니다.
1.  왼쪽 메뉴 **App Hosting** 클릭
2.  목록에 있는 백엔드(예: `teen-qt-backend`) 클릭
3.  상단 메뉴 중 **[설정(Settings)]** 클릭
4.  화면 맨 아래로 내려가서 **[백엔드 삭제(Delete backend)]** 클릭
5.  삭제가 완료되면 다시 **[새 백엔드 만들기]** 버튼을 눌러 처음부터 진행하세요.

---

## 🚨 [저장]이나 [취소] 버튼이 안 눌릴 때 (긴급!)
1.  **페이지 새로고침 (F5)**: 가장 먼저 해보세요! 꼬인 화면이 풀립니다.
2.  **시크릿 모드 사용**: 키보드에서 `Ctrl + Shift + N`을 눌러 시크릿 창을 열고 다시 시도해 보세요. (브라우저 오류 해결)
3.  **환경 이름 다시 쓰기**: `prod`라고 썼을 때 버튼이 안 눌리면, 다 지우고 `production`이라고 길게 써보세요. 파란색 버튼이 활성화되는 순간 클릭!
4.  **목록 클릭**: 이미 저장된 것 같다면, 화면 아래쪽에 생긴 `prod` 또는 `production`이라는 **글자 자체를 클릭**해야 상세 메뉴(환경 변수)가 나옵니다.

---

## 1. 환경 변수(API 키) 등록 핵심 요약
1.  상단 메뉴의 **[설정(Settings)]** 클릭
2.  왼쪽 메뉴의 **[환경(Environment)]** 클릭
3.  목록에 있는 `prod` (또는 본인이 만든 이름) 클릭
4.  **[환경 변수(Environment variables)]** 섹션에서 **[Add variable]** 클릭
    *   **Key**: `GEMINI_API_KEY`
    *   **Value**: 본인의 AI API 키 붙여넣기
5.  하단의 **[저장]** 클릭

## 2. 빌드 오류 해결하기
환경 변수를 등록한 후에는 반드시 다시 빌드해야 합니다.
1.  상단의 **[출시(Release)]** 탭 클릭
2.  우측 상단의 **[출시 만들기(Create release)]** 클릭
3.  이제 API 키가 인식되어 정상적으로 배포가 완료됩니다!
